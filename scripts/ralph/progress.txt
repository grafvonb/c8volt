## Codebase Patterns
- For versioned internal service refactors, mirror the `resource` and `processdefinition` layout: package `api.go` owns the shared interface assertions, each versioned `contract.go` declares a matching local `API`, and generated-client interfaces should only list endpoints that the versioned service actually calls.
- For versioned internal services, build constructors with `internal/services/common.PrepareServiceDeps` and finish with `common.EnsureLoggerAndClients` so nil-safe defaults and test-time client injection stay aligned across packages.
- Camunda v8.8 generated search request tests should assert serialized flat equality values like `"tenantId":"tenant"` instead of expecting `"$eq"` wrappers; the generated request unions flatten simple equality filters during JSON marshaling.
- Processinstance walker `Descendants` results are root-inclusive and keep explicit `nil` edge entries for leaves; preserve that shape because higher-level delete and tree-rendering flows depend on it.
- For internal service generated-client coverage stories, finish the review in `research.md` with an explicit candidate-by-candidate keep/defer decision; if no endpoint is added, document the boundary reasons instead of leaving the omission implicit.
- Camunda v8.7 processinstance create responses do not expose the created instance key, so wait-oriented regression coverage should use delete/state-polling paths rather than assuming create-time confirmation can start from the create payload.
