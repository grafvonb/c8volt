## Codebase Patterns
- `cmd` completion regressions should execute Cobra's hidden `__complete` or `__completeNoDesc` root entry points after resetting shared flags; this exercises the real completion seam without needing shell wrappers.
- Completion regressions should assert the same forbidden-output contract on flag-value completion paths as on command completion paths so helper names or usage/help text cannot leak through narrower suggestion cases.
- Setting `rootCmd.CompletionOptions.DisableDescriptions = true` shifts generated shell scripts onto Cobra's `__completeNoDesc` path, which is the stable way to keep interactive completion output name-only without forking Cobra.
- Use `rootCmd.CompletionOptions.HiddenDefaultCmd = true` when Cobra's generated `completion` command should stay executable for script generation but must not appear in normal help or interactive completion suggestions.
- With root completion descriptions enabled, Cobra uses each command's concise `Short` text for interactive suggestions while custom flag-value completions that return plain strings still render without tab-separated descriptions.
- When invoking Cobra's hidden `__complete` or `__completeNoDesc` commands directly, pass arguments relative to the root command and omit the `c8volt` binary name or Cobra will treat the request as an invalid command path.
