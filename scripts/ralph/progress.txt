## Codebase Patterns
- Public `c8volt/process` facade methods should delegate to `internal/services/processdefinition` and map facade options with `options.MapFacadeOptionsToCallOptions` instead of adding command-specific service paths.
- Generated XML responses can populate `XML200` with an empty string even when `Body` contains the BPMN XML; versioned XML service methods should validate via `common.RequirePayload` and then fall back to the raw response body when needed.
- `get process-definition --xml` is a dedicated raw-output path: it must reject global render flags like `--json`/`--keys-only` and extra process-definition filters instead of silently mixing XML mode with list/detail rendering behavior.
- Generated CLI reference pages come from Cobra command metadata such as `Long`, `Example`, and flag help text; update the command definition first and regenerate with `make docs` instead of hand-editing `docs/cli`.
