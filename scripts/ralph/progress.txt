# Ralph Progress Log
## Codebase Patterns
- Resource version packages should mirror the shared service surface with a local `API` interface and compile-time `var _ API = (*Service)(nil)` assertion, while `contract.go` keeps only the generated-client methods the service actually uses.
- Service factory tests in this repo should assert the concrete versioned service type for each supported Camunda version and verify unsupported versions with `services.ErrUnknownAPIVersion`, including the normalized `"unknown"` text and supported-version list.
- Resource deploy unit tests can stay fast and deterministic by mocking the generated client and parsing the multipart request body directly to verify tenant and `resources` form fields.
- For v8.8 resource confirmation coverage, mock the process-definition generated client and exercise `processDefinitionDeployPoller` directly for wait-state vs success semantics instead of relying on the real poller backoff loop.
- Cross-version resource metadata lookup should map each generated `ResourceResult` into `internal/domain.Resource` and continue to use `common.RequirePayload` for malformed-success protection; defer raw content retrieval unless there is a caller that justifies widening the API.
- Resource delete regression tests should cover both the default consistent no-op path and the `WithAllowInconsistent()` path propagating mapped HTTP status errors from the generated deletion endpoint.
