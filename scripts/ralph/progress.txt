## Codebase Patterns
- Keep `internal/services/cluster/api.go` and each version package `contract.go` on the same method set, with compile-time interface assertions in both places to catch drift before refactors land.
- Put the shared cluster topology fetch/log/validation flow in `internal/services/cluster/common`, and keep version packages focused on generated-client calls plus payload conversion.
- When extending cluster capabilities, prefer endpoints that exist in every supported generated Camunda client and normalize version-specific optional fields in `internal/domain/cluster.go` before exposing them through `c8volt/cluster`.
- Integration helpers should call facade methods with the current signatures; resource deployment uses `cfg.App.Tenant` from config, so `DeployProcessDefinition` takes deployment units plus options, not an explicit tenant argument.
- Command help tests can execute the shared root command directly with `SetArgs`, `SetOut`, and `ExecuteC` to verify Cobra tree wiring without standing up service dependencies.
- For CLI command tests that execute real handlers, write an explicit temp `--config` file instead of relying on ambient env or repo-local config, and use a subprocess helper when you need to assert `os.Exit` paths.
- When `cmd` tests reuse `Root()` in-process, reset Cobra flag state between executions; help runs leave flags set on the shared command tree and can make later command executions show help unexpectedly.
