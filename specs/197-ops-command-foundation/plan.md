# Implementation Plan: Ops Command Foundation

**Branch**: `197-ops-command-foundation` | **Date**: 2026-05-10 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/197-ops-command-foundation/spec.md`

**Mandatory Implementation Context**: Ralph and any implementation agent MUST receive `--implementation-context specs/ralph-implementation-rules.md`.

## Summary

Add the `c8volt ops` command family as grouping-only CLI structure for future operational workflows. The implementation should add `ops`, `ops execute`, and `ops repair` commands using the existing Cobra command patterns, expose command contract metadata for discovery, define lightweight shared workflow/report status contracts only where immediately useful, add focused command tests, and regenerate generated CLI documentation. Concrete ops playbooks remain out of scope.

## Technical Context

**Language/Version**: Go, using the repository's current Go toolchain
**Primary Dependencies**: Cobra CLI commands, existing command metadata helpers, existing docs generator
**Storage**: N/A; no persistent data model
**Testing**: Go tests in `cmd` plus generated docs validation where relevant
**Target Platform**: CLI on supported local/operator environments
**Project Type**: Go CLI application
**Performance Goals**: Help and discovery commands remain immediate and do not initialize Camunda runtime clients for grouping help paths
**Constraints**: Preserve existing command behavior; do not implement concrete ops playbooks; keep automation JSON stdout deterministic for future workflow contracts; follow `specs/ralph-implementation-rules.md`
**Scale/Scope**: Three grouping commands, shared contract/status definitions, focused tests, generated documentation
